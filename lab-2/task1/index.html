Примеры параллельных вычислений

Три примера реализации для вычисления суммы чисел в заданном диапазоне параллельно, используя три разных подхода: Asyncio, Multiprocessing и Threading.

Asyncio

import asyncio
import time

async def calculate_sum(start, end):
    total = 0
    for i in range(start, end+1):
        total += i
    return total

async def main():
    start_time = time.time()

    chunk_size = 10_000_000
    num_tasks = 4

    tasks = [asyncio.create_task(calculate_sum(i * chunk_size + 1, (i + 1) * chunk_size)) for i in range(num_tasks)]
    results = await asyncio.gather(*tasks)

    print(sum(results))

    end_time = time.time()
    print(f"Время выполнения: {end_time - start_time:.2f} секунд")

if __name__ == "__main__":
    asyncio.run(main())

Этот пример использует модуль asyncio и асинхронный подход для параллельного вычисления суммы чисел в заданном диапазоне. Функция calculate_sum() является асинхронной функцией, которая вычисляет сумму в определенном диапазоне. В функции main() создаются четыре отдельные асинхронные задачи и запускаются параллельно с помощью asyncio.gather(). После завершения всех задач, результаты суммируются и время выполнения выводится на печать.

Multiprocessing

import multiprocessing
import time

def calculate_sum(start, end):
    total = 0
    for i in range(start, end+1):
        total += i
    return total

def main():
    start_time = time.time()

    processes = []
    chunk_size = 10_000_000
    num_processes = 4

    for i in range(num_processes):
        start = i * chunk_size + 1
        end = (i + 1) * chunk_size
        process = multiprocessing.Process(target=lambda: print(calculate_sum(start, end)))
        process.start()
        processes.append(process)

    for process in processes:
        process.join()

    end_time = time.time()
    print(f"Время выполнения: {end_time - start_time:.2f} секунд")

if __name__ == "__main__":
    main()

Этот пример использует модуль multiprocessing для параллельного вычисления суммы чисел в заданном диапазоне. Функция calculate_sum() вычисляет сумму в определенном диапазоне. В функции main() создаются четыре отдельных процесса и запускаются параллельно. После завершения всех процессов, время выполнения выводится на печать.

Threading

import threading
import time

def calculate_sum(start, end):
    total = 0
    for i in range(start, end+1):
        total += i
    return total

def main():
    start_time = time.time()

    threads = []
    chunk_size = 10_000_000
    num_threads = 4

    for i in range(num_threads):
        start = i * chunk_size + 1
        end = (i + 1) * chunk_size
        thread = threading.Thread(target=lambda: print(calculate_sum(start, end)))
        thread.start()
        threads.append(thread)

    for thread in threads:
        thread.join()

    end_time = time.time()
    print(f"Время выполнения: {end_time - start_time:.2f} секунд")

if __name__ == "__main__":
    main()

Этот пример использует модуль threading для параллельного вычисления суммы чисел в заданном диапазоне. Функция calculate_sum() вычисляет сумму в определенном диапазоне. В функции main() создаются четыре отдельных потока и запускаются параллельно. После завершения всех потоков, время выполнения выводится на печать.

Различия

  1. Asyncio: Использует асинхронный подход с использованием корутин и цикла событий. Это подходит для задач, ориентированных на ввод-вывод.
  2. Multiprocessing: Использует отдельные процессы для параллельного выполнения задач. Это подходит для задач, ориентированных на процессор, так как может использовать несколько ядер процессора.
  3. Threading: Использует потоки для параллельного выполнения задач. Это подходит для задач, ориентированных на ввод-вывод, так как потоки могут переключаться на другие задачи при ожидании ввода-вывода.

Время выполнения

Подход Время выполнения (секунды)
Asyncio 1.03 seconds
Multiprocessing 0.401897668838501 seconds
Threading 0.93 seconds

Из результатов видно, что наилучшую производительность показал подход Multiprocessing, затратив всего 0.401897668838501 секунд на выполнение. Это связано с тем, что задача является CPU-bound, и Multiprocessing может эффективно использовать несколько ядер процессора для параллельной обработки.
