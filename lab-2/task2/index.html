Параллельное сохранение данных из веб-страниц

Три примера реализации для параллельного извлечения данных с нескольких URL-адресов и их сохранения в базу данных, используя три разных подхода: Asyncio, Multiprocessing и Threading.

Asyncio

import asyncio
import aiohttp
from bs4 import BeautifulSoup
import psycopg2
import re
import time

urls = [
    'https://career.habr.com/vacancies?s%5B%5D=2&s%5B%5D=3&s%5B%5D=82&s%5B%5D=4&s%5B%5D=5&s%5B%5D=72&s%5B%5D=1&s%5B%5D=75&s%5B%5D=6&s%5B%5D=77&s%5B%5D=7&s%5B%5D=83&s%5B%5D=84&s%5B%5D=8&s%5B%5D=85&s%5B%5D=73&s%5B%5D=9&s%5B%5D=86&s%5B%5D=106&type=all',
    'https://career.habr.com/vacancies?s[]=2&s[]=3&s[]=82&s[]=4&s[]=5&s[]=72&s[]=1&s[]=75&s[]=6&s[]=77&s[]=7&s[]=83&s[]=84&s[]=8&s[]=85&s[]=73&s[]=9&s[]=86&s[]=106&sort=salary_desc&type=all',
    'https://career.habr.com/vacancies?s[]=1&s[]=75&s[]=6&s[]=77&s[]=7&s[]=83&s[]=8&s[]=85&s[]=73&s[]=9&s[]=86&sort=date&type=all&with_salary=true',
    'https://career.habr.com/vacancies?locations[]=c_699&s[]=1&s[]=75&s[]=6&s[]=77&s[]=7&s[]=83&s[]=8&s[]=85&s[]=73&s[]=9&s[]=86&sort=date&type=all&with_salary=true'
]

conn = psycopg2.connect(
    host="localhost",
    database="finance_db",
    user="postgres",
    password="1234"
)
c = conn.cursor()

async def parse_and_save(url):
    start_time = time.time()
    async with aiohttp.ClientSession() as session:
        async with session.get(url) as response:
            html = await response.text()
            soup = BeautifulSoup(html, 'html.parser')
            vacancies = soup.find_all('a', {'class': 'vacancy-card__title-link'})
            salaries = soup.find_all('div', {'class': 'basic-salary'})
            for idx, vac in enumerate(vacancies):
                if salaries[idx].text:
                    s = salaries[idx].text
                else:
                    s = 100000
                print(vac.text, s)
                c.execute("INSERT INTO financialtransaction (description, amount, user_id) VALUES (%s, %s, %s)", (str(vac.text), re.findall(r'\d+', str(s))[0], 2))
            title = soup.title.string

            conn.commit()
            print(f"Сохранен титул '{title}' из {url}")
            end_time = time.time()
            print(f"Время парсинга: {end_time - start_time:.2f} секунд")


async def main():
    await asyncio.gather(*[parse_and_save(url) for url in urls])

if __name__ == '__main__':
    asyncio.run(main())
    conn.close()
Этот пример использует модули asyncio и aiohttp для параллельного извлечения данных с нескольких URL-адресов. Функция parse_and_save() является асинхронной функцией, которая извлекает данные из каждого URL-адреса, анализирует HTML-страницу и сохраняет данные в базу данных Postgres. В функции main() все асинхронные задачи выполняются параллельно с помощью asyncio.gather(). После завершения всех задач, соединение с базой данных закрывается.

Этот пример использует модуль threading для параллельного извлечения данных с нескольких URL-адресов. Функция parse_and_save() извлекает данные из каждого URL-адреса, анализирует HTML-страницу и сохраняет данные в базу данных Postgres. В функции main() создаются отдельные потоки для обработки каждого URL-адреса, и они запускаются параллельно. После завершения всех потоков, соединение с базой данных закрывается.

Различия

  1. Asyncio: Использует асинхронный подход с использованием корутин и цикла событий. Это подходит для задач, ориентированных на ввод-вывод, так как может переключаться между задачами при ожидании ввода-вывода.
  2. Multiprocessing: Использует отдельные процессы для параллельного выполнения задач. Это подходит для задач, ориентированных на процессор, так как может использовать несколько ядер процессора.
  3. Threading: Использует потоки для параллельного выполнения задач. Это подходит для задач, ориентированных на ввод-вывод, так как потоки могут переключаться на другие задачи при ожидании ввода-вывода.

Время выполнения

Подход Время выполнения (секунды)
Asyncio 3.89 секунд
Multiprocessing 1.20 секунд
Threading 1.17 секунд

В этом случае результаты отличаются. Подход Threading показал лучшую производительность, затратив 1.17 секунд на выполнение. Multiprocessing занял второе место, затратив 1.20 секунд.

Такие результаты можно объяснить тем, что во втором задании задача является более I/O-bound, поэтому Threading и Multiprocessing оказались более эффективными, чем Asyncio. Asyncio, хотя и эффективен для I/O-bound задач, в данном случае уступил по производительности двум другим подходам.
